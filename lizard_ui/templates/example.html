About the bar

The bar above the sidebar and this content area is divided into three areas:

  1. The sidebar header.
  2. A couple of "object tabs": click them and you get a different view for the currently viewed object. Object is kinda loose: it can be a selected area or a specific report or the homepage. But for a selected area you might want to show an overview with graphs, a table with raw data and an edit form. That's where "object tabs" are for.
  3. A couple of "object actions". Terminology is a bit borrowed from Plone (big python CMS) where they've really nailed down the concept of a pluggable interface, btw. Actions are things like "print" and "export" (and perhaps "edit", though that can be a tab, too).

About blueprint's advantage

A big big big advantage is that a css framework like blueprint is cross-browser. No more weird IE bugs that come to bite you. Consistency across browsers. Huge timesaver, especially as we don't have a real css expert as far as I know.

A second advantage is that you get a reliable, known layout. You don't have to worry how wide or narrow something ends up as, you just define it. So for graphs you just *know* you can make them 500px (or so) wide and they'll fit beautifully.

You need less css. The basics and IE corner cases are all taken care off, so most html elements just work like you want them to. The "only" things you need to do yourself, typically, are background colors, GUI items like tabs and application-specific tweaks.

As long as you stick to blueprint's way of lining things out, the typography is pretty OK. Line height, margins and font sizes all match. No more one-pixel size differences that can make your website look bad.

Drawback: you're tied to the framework. You've got to work with it. Which can be a good thing anyway. One specific drawback is that the layout is fixed at a 950px width. This helps designing a layout that also works on the smaller monitors of our customers, but it doesn't take advantage of our own big monitors. (Note that there are extensions that make the framework "fluid" so that it stretches, but I haven't used those yet).
